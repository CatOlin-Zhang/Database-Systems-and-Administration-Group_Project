# Database-Systems-and-Administration-Group_Project
核心功能模块 
• 供应商：平台上的每个供应商都将拥有专属档案，包含供应商ID、企业名称、客户平均评分、地域分布及产品目录。 
• 产品：每件商品均通过唯一产品ID标识，包含名称、标价、库存数量及最多三个由销售方设置的产品属性标签。商品可关联多个订单。 
• 客户：需维护完整客户数据库，每位客户拥有唯一客户ID、联系电话、收货地址及订单历史记录。客户可发起多笔订单。 
• 订单：每笔订单包含唯一订单ID、客户ID、下单日期、商品清单及数量、总价和状态信息。单笔订单可能涉及跨供应商交易。 
• 交易记录：所有采购交易均关联客户订单、客户向供应商付款记录，并包含供应商产品目录中的至少一件商品。

功能模块 
• 供应商管理：平台需具备以下功能：
    1. 显示所有供应商名录；2. 实现新供应商入驻市场。 
• 产品目录管理：用户可执行以下操作：
    1. 浏览各供应商提供的全部产品；2. 将新产品添加至供应商目录。 
• 产品发现功能：
    系统需提供基于标签的搜索功能，支持用户通过标签匹配产品名称或关联标签来查找商品。
    搜索结果应包含标签与产品名称任意部分或关联标签完全匹配的产品。 
• 产品采购：系统需支持产品购买功能，并在数据库中记录客户购买的具体商品信息。 
• 订单修改：用户应具备订单修改权限，包括在订单进入物流环节前删除特定商品或取消整个订单。

项目作业要求：
强烈建议使用Java 11及以上版本和Python 3.5及以上版本，其他版本可能无法在部门计算机上运行并影响评分。
请分别通过JDBC和PyMySQL连接MySQL数据库。
数据库设计与系统实现 
1. 每个小组需设计ER图并将其转换为关系表结构。
2. 每个小组需实现系统所需功能模块。
3. 每个小组需提交最终报告的电子版及实现代码的电子版文档。
4. 每个小组需编写readme文件说明代码运行方法。
5. 每个小组需录制不超过6分钟的视频，向课程讲师（客户方）展示电商平台功能，内容需包含ER图设计、关系表结构设计及功能演示过程。

正确性（40%）：若实现完整且所有功能运行正确、规范且稳健，可获得满分。部分正确提交将获得部分分数。
视频演示（40%）：评分标准如下：
    1. 演示清晰度（解释ER图、关系表及功能时）（20%）
    2. 语言表达能力（10%）
    3. 演示功能数量（10%）注：各小组该部分总分将乘以系数（3-x）÷3，其中x为视频时长超过6分钟的分钟数。若视频时长≥9分钟，该小组本部分得分为零。
文档说明（20%）：报告应简明扼要、清晰易懂且信息充分。
